Sim, em ambas as soluçõs (Python e C), as threads compartilham um array de maiores notas por classe entre si, indexados pelo identificador da classe - sabendo que cada classe terá suas notas avaliadas por uma thread distinta. Embora compartilhem este array para armazenamento de dados entre si, cada thread acessa apenas o espaço cujo índice equivale ao seu identificador.
Isto evita uma condição de corrida pois não temos múltiplas thread atualizado exatamente o mesmo espaço na memória - embora estejam compartilhando o mesmo array entre si.
Para o objetivo desejado, a solução desenvolvida é satisfatória. É instanciado um array global para armazenamento desses dados, e este array é acessado e atualizado em espaços distintos a depender da thread filha. Com isto, ao finalizar o processamento de todas as threads criadas, a thread mãe possui as informações relevantes produzidas por cada uma de suas filhas, como a maior nota de cada classe.
